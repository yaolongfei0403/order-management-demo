# 秩云AI Factory OS V3.0 优化需求清单
## 基于评审意见与行业最佳实践的HTML页面优化方案

---

## 一、设计原则（新增）

### 1.1 AI-Native设计范式
- **目标驱动（Goal-Driven）**：用户打开系统看到的不是功能菜单，而是"今天要完成什么"
- **Agent-First**：AI Agent是主要用户，人类是监督者（Supervisor）
- **闭环优先**：每个业务问题必须有"发现→方案→执行→验证→学习"的完整链路
- **渐进式披露**：从简洁的任务卡片开始，按需展开详情

### 1.2 参考行业趋势
- Microsoft Copilot 2026新设计：将静态文本框转变为"任务感知工作区"，支持渐进式披露
- Agentic AI UX设计：用户从"操作者"变为"监督者"，需要实时可见性和控制权
- AI-Native ERP架构：Agent直接读写数据，而非通过UI点击
- Command Center模式：可视化Agent协作、升级和工作交接

## 三、Today Mission页面重构（P0）

### 3.1 当前问题
- 页面是Dashboard，展示KPI+AI员工状态+数据快照
- 用户需要"理解数据→判断→决策→行动"，认知负荷高
- 没有体现"目标驱动"的AI-Native范式

### 3.2 重构目标
将首页从"数据展示"变为"任务驱动"，用户打开系统直接看到"今天要完成什么"。

### 3.3 页面结构（HTML实现）

核心结构：三栏式任务看板
- 左栏：进行中（AI处理中，需人工确认）
- 中栏：待开始（AI已规划，等待触发）
- 右栏：已完成（已闭环，经验沉淀）

每个任务卡片必须包含：
1. **目标描述**（如"订单HT-001风险解除"）
2. **AI执行进度条**（0%-100%）
3. **当前状态**（AI处理中/等待人工审批/整改中）
4. **行动按钮**（查看详情/一键批准/查看进度）
5. **AI置信度**

### 3.4 关键交互要求
- 一键批准：点击后直接执行AI方案，无需跳转其他页面
- 任务卡片hover展开详情
- 进行中任务实时更新进度

## 四、AI执行中心新增（P0）

### 4.1 页面定位
这是评审认为**最缺失的核心页面**，当前原型完全没有。这是区分"带Agent的ERP"和"AI Factory OS"的关键。

### 4.2 页面结构

核心结构：Tab切换 + 闭环任务卡片

**Tab1：执行中**
- 展示AI正在自动执行的闭环任务
- 每个任务包含6步流程可视化：发现→分析→方案→审批→执行→验证
- 当前步骤高亮显示
- 显示执行进度和预计完成时间

**Tab2：待确认**
- 展示需要人工审批的AI方案
- 每个方案包含：问题描述+根因分析+推荐方案+成本估算+预期效果
- 提供"批准"/"修改"/"驳回"按钮

**Tab3：已完成**
- 展示已闭环的任务
- 包含：实际结果+效果评价+经验沉淀状态
- 支持按时间/效果/Agent筛选

### 4.3 闭环流程可视化

每个任务卡片顶部展示6步流程：
```
[发现] → [分析] → [方案] → [审批] → [执行] → [验证]
  ✓        ✓        ✓        ◐        ○        ○
```
- 已完成步骤：绿色✓
- 当前步骤：蓝色高亮+动画
- 待完成步骤：灰色○

## 五、数字员工工作台升级（P0）

### 5.1 当前问题
- 像"在线状态面板"，展示"工作中/待命/告警"
- 缺少"AI干了什么"的成果展示

### 5.2 升级为Agent Mission Center

每个Agent卡片必须包含：
1. **🎯 本月目标**（如"确保订单准时率>95%"）
2. **📋 当前负责**（如"14个订单"）
3. **🔄 正在执行**（进度条+任务描述）
4. **⏳ 等待确认**（待审批事项数）
5. **🏆 本月贡献**（追回延期次数/节省工时/产能利用率提升）
6. **📈 准确率趋势**（当前准确率+环比变化）

### 5.3 关键变化
- 从"AI在线"变成"AI完成了什么、创造了什么价值"
- 每个Agent都有明确的KPI目标
- 展示Agent对业务的实际贡献

## 六、Copilot升级为Command Center（P1）

### 6.1 当前问题
- 聊天问答模式，返回一大段文字
- 用户需要阅读理解后再去其他模块操作

### 6.2 升级为AI Command Center

AI回复结构改为：
1. **📊 分析结果**（问题概述）
2. **问题卡片**（具体问题+根因+置信度）
3. **💡 AI推荐最优方案**（高亮推荐）
4. **执行按钮**（立即执行/查看所有方案/忽略）

关键变化：不是"告诉你问题"，而是"告诉你问题+方案+执行按钮"。

### 6.3 快捷指令升级
- "我有哪些订单有延期风险？" → 返回分析+方案+执行按钮
- "帮我生成采购计划" → 返回计划+一键生成采购单按钮
- "分析本月利润下降原因" → 返回归因+优化方案+执行按钮

## 八、预警中心增加闭环追踪（P1）

### 8.1 当前问题

```javascript

  discoveredAt: '2026-06-29T08:30:00',    // 发现时间
  analyzedAt: '2026-06-29T08:31:00',        // AI分析时间

  approvedAt: null,                          // 审批时间
  executedAt: null,                          // 执行时间
  verifiedAt: null,                        // 验证时间
  learnedAt: null                          // 经验沉淀时间
}
```

### 8.3 预警列表增加状态筛选
- 未处理
- AI分析中
- 方案待审批
- 执行中
- 待验证
- 已闭环
- 已沉淀
