## 1. 产品概述

### 1.1 一句话定位
基于大模型与企业知识库构建的 **AI Factory OS（AI工厂操作系统）**，通过数字员工、智能决策 Agent 与自然语言交互，实现从订单接收、生产排程、采购协同到经营分析的全流程自动化管理。

### 1.2 核心范式转变

| 维度 | 传统 ERP/MES | AI Factory OS |
|---|---|---|
| **交互方式** | 人找功能 → 点菜单 → 填表单 | 描述目标 → AI理解 → 自动执行 → 人确认 |
| **数据关系** | 数据分散在各模块，人做关联 | AI跨模块自动关联分析 |
| **决策模式** | 人做判断，系统记录 | 系统主动发现问题并给出方案 |
| **知识沉淀** | 依赖人工文档和经验传承 | AI自动萃取工艺DNA，越用越聪明 |
| **系统角色** | 记录工具 | 数字员工团队 |

### 1.3 关键指标（演示态）

### 1.4 核心价值主张

1. **4个AI数字员工** — 不是买了一套系统，而是雇了4个7×24小时工作的智能员工。
2. **自然语言驱动** — 用说话代替点菜单，用目标描述代替流程操作。

## 4. 第二层：AI 大脑层

### 4.1 智能工厂数字孪生引擎（底座）

**定位**：所有AI决策的数据基础和仿真环境。

**4.1.1 实时数字镜像**
- **数据融合**：IoT设备数据 + 人工录入 + 外部数据（天气、面料期货价格）
- **镜像精度**：生产状态延迟 < 30秒，库存状态延迟 < 5分钟
- **可视化**：3D工厂模型，可下钻到产线 → 设备 → 工单

**4.1.2 预测仿真沙盒（What-if）**
- 厂长可以问："如果下周接3个急单，现有产能会怎样？"
- AI运行仿真：
  ```
  输入：3个急单（各5000件，7天交付）
  仿真结果：
  ├─ 产能缺口：缝制工序缺口30%
  ├─ 物料风险：面料库存不足，需紧急采购
  ├─ 延期概率：85%（至少1单延期）
  ├─ 最优方案：外协30%缝制产能 + 紧急采购面料
  └─ 成本增加：¥45,000
  ```

**4.1.3 因果推理引擎**
- 不是"库存低了"，而是：
  ```
  库存不足根因链：
  订单HT-2026-0615-001加急（客户临时提前交期）
  → 领料量超出计划（+40%）
  → 面料库存快速消耗
  → 采购周期未调整（仍按原计划15天）
  → 库存低于安全线

  最优解法：
  ① 紧急采购（2天到货，+¥2,400）
  ② 从订单HT-2026-0620-005调剂（该订单不紧急，-¥0）
  ③ 与客户协商延期2天（-客户满意度5%）

  AI推荐：方案②，零成本，不影响紧急订单
  ```

**4.1.4 自学习优化器**
- **决策记录**：AI每次建议记录完整上下文（输入/输出/用户选择）
- **效果追踪**：追踪采纳建议的实际结果（按时交付？成本如何？）
- **模型微调**：每周自动用新数据微调，准确率持续提升
- **知识蒸馏**：单厂经验脱敏后进入行业通用模型

### 4.2 AI Agent 决策执行层

#### 4.2.1 采购Agent（P0）

**自主工作流**：
```
监控库存 → 预测消耗 → 生成采购建议 → 供应商比价 → 发起审批 → 跟踪到货 → 入库确认
```

**触发条件**：
- 库存低于安全线（自动）
- 新订单BOM拆解后物料缺口（自动）
- 面料价格波动超过阈值（自动）
- 用户语音指令（"帮我采购黑色面料"）

**决策能力**：
- **供应商选择**：综合考虑历史质量、价格、交期、当前产能负载
- **批量优化**：合并多个订单的相同物料采购，争取批量折扣
- **价格锁定**：检测到价格上涨趋势时，建议提前锁定长期合同

**人机协同点**：
- Agent生成采购建议 → 推送待确认 → 用户一键确认/修改/驳回
- 紧急采购（>¥10,000）必须人工确认
- 常规采购（<¥10,000）可设置自动执行

#### 4.2.2 排产Agent（P0）

**自主工作流**：
```
接收订单 → 分析BOM和工艺路径 → 评估产能负载 → 生成排产方案 → 模拟验证 → 发布执行 → 实时监控 → 动态调整
```

**核心能力**：
- **多目标优化**：交期满足率 + 设备利用率 + 人工成本 + 换线次数
- **动态重排**：检测到异常（设备故障/物料延迟/人员变动）时，自动重算并推送调整方案
- **瓶颈预测**：提前3天预测产能瓶颈，给出预防方案

**示例交互**：
```
AI："检测到下周缝制B组有2天设备保养计划，已自动调整排产：
   ① 将订单HT-2026-0625-002提前到本周五开工
   ② 订单HT-2026-0628-004延后2天，已与客户沟通确认
   ③ 建议外协2000件给佛山金辉，预计增加成本¥3,200
   确认执行吗？"

用户："确认"
AI："已执行。相关订单状态已更新，客户通知已发送。"
```

#### 4.2.3 交期Agent（P0）

**自主工作流**：
```
监控所有在制订单 → 预测完成时间 → 识别延期风险 → 根因分析 → 生成补救方案 → 通知相关人员
```

**预警等级**：
- 🔴 紧急（<1天）：立即通知厂长 + 生成紧急方案
- 🟠 高（1-3天）：通知计划员 + 生成调整建议
- 🟡 中（3-7天）：系统记录 + 纳入排产优化

**补救方案自动生成**：
```
订单HT-2026-0615-001 预计延期3天

根因分析：
├─ 直接原因：面料延迟到货2天
├─ 深层原因：供应商A产能饱和（本月订单↑40%）
```
